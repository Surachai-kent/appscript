3. Other handler operations

Removing event handlers

removeeventhandler(eventcode)
    
    eventcode : str -- 8-letter code indicating the Apple event handler's 
                       event class and id

Installing coercion handlers

aemreceive allows Python functions to be installed as coercion handlers, augmenting/replacing the Apple Event Manager's built-in coercion handlers.

The installer function

installcoercionhandler(coercioncallback, fromtype, totype)

    coercioncallback : function -- the callback function invoked to 
                                   process the Apple event
                                   
    fromtype : str -- the four-character descriptor type to coerce from
    
    totype : str -- the four-character descriptor type to coerce to

The coercion callback

installcoercionhandler takes a callback function as its first argument. The signature for this callback is:

coercioncallback(value, totype)
    
    value : AEDesc -- the descriptor containing the data to coerce
    
    totype : str -- the desired OSType for the new descriptor
    
    Result : AEDesc -- the new descriptor

If the callback raises an error, a coercion error (-1700) is automatically returned to the calling application.

Removing coercion handlers

removecoercionhandler(fromtype, totype)
                                   
    fromtype : str -- the four-character descriptor type to coerce from
    
    totype : str -- the four-character descriptor type to coerce to
