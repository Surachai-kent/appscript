Installing PyOSA dependencies

PyOSA requires py-appscript 0.18.0 or later to be installed before use. Binary and source distributions can be downloaded from the appscript website (http://appscript.sourceforge.net). Because PyOSA dynamically loads a pre-existing Python framework instead of providing its own, it is recommended that appscript be installed for all publically available Python frameworks (i.e. the Apple-installed Python framework and any user-installed Python frameworks). See the appscript documentation for further instructions.

Installing the PyOSA component

To install PyOSA, copy the PyOSA.component bundle to /Library/Components or ~/Library/Components. (You may need to create a Components folder first if one doesn't already exist.) OSA-aware applications such as Script Editor will detect PyOSA the next time they are launched.
